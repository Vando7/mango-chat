# Agent Guidelines тАФ chat-app

## Rules
- **Always keep this file up to date.** Every time you modify code, configs, or architecture, update the relevant section below. Outdated docs cause mistakes.
- Follow the file structure described below. New files go in `src/` under `api/` or `components/`.
- Use TypeScript/JSX consistently. No new .ts files unless explicitly requested.
- Run `npx vite build` to verify before confirming changes work.

## Architecture
- **`src/App.jsx`** тАФ Main app component, manages all state and orchestration
- **`src/api/client.js`** тАФ API client (`fetchModels`, `chat`, `setApiBase`)
- **`src/components/MessageList.jsx`** тАФ Message rendering & auto-scroll
- **`src/components/ChatInput.jsx`** тАФ Text input, image upload, send button
- **`src/components/SettingsPanel.jsx`** тАФ Server URL & model selector
- **`vite.config.js`** тАФ Vite dev server with `/v1` proxy to `localhost:13305`
- **`tailwind.config.js`** тАФ Tailwind CSS config
- **`dev.sh`** тАФ Simple dev runner script

## Tech Stack
- React 18 + Vite 8
- Tailwind CSS 3
- Vite dev proxy for CORS-free API calls

## Common Patterns
- Messages are `{ role, content, image?, streaming? }`
- Image content uses OpenAI multimodal format: `{ type: 'image_url', image_url: { url: base64 } }`
- Streaming uses `async function*` generators, consumed with `for await`
- Error messages appear as red text in SettingsPanel

## Testing (for the agent)
```bash
cd chat-app

# Check if server is reachable + list models
bash scripts/check.sh

# Test streaming with a message
node scripts/test-stream.js "user.Qwen3.6-35B-A3B-GGUF-UD-Q2_K_XL" "Explain quantum entanglement in 3 sentences"

# Test non-streaming chat
bash scripts/test-api.sh

# Test image upload
node scripts/test-image.js "user.Qwen3.6-35B-A3B-GGUF-UD-Q2_K_XL" "Describe this" ./test.png

# Or use npm scripts
npm run check
npm run test:stream
npm run test:api
```
